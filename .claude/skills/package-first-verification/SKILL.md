# Skill: Package-First Verification

## Purpose
Enforce the package-first architecture by ensuring every implementation begins with a proprietary package scan. The checklist at `foundation_documentation/package_registry.md` is auto-generated by `delphi-ai/tools/verify_package_registry.sh` and uses checkboxes to indicate usage status.

## When to Invoke
- Before planning implementation of any new feature.
- Before creating a new controller, service, repository, utility, or helper.
- Before adding a third-party dependency.
- When a TODO involves creating new files in `app/Services/`, `app/Helpers/`, `lib/utils/`, or similar host-level utility paths.

## Procedure

### 1. Read the Checklist
Open `foundation_documentation/package_registry.md`. If it does not exist or is stale, run:
```bash
bash delphi-ai/tools/verify_package_registry.sh --project-root <path>
```

### 2. Interpret the Checkboxes

| Status | Meaning | Agent Behavior |
| :--- | :--- | :--- |
| `[x]` package-name — path (in use) | Package is a declared dependency | **Use it.** Read its README, understand its API, extend if needed. |
| `[ ]` package-name — path (available) | Package exists but is NOT a dependency | **Recommend adoption.** Read its README, evaluate if it covers the planned work, suggest integration. |

### 3. Read the README
For each relevant proprietary package (whether `[x]` or `[ ]`), read its `README.md` to understand:
- What the package does (purpose)
- Public API / key exports
- How to integrate / extend

The README is the authoritative documentation — not the checklist.

### 4. Evaluate and Decide

| Situation | Action |
| :--- | :--- |
| `[x]` package covers the need | **Use directly.** Import and call its API. |
| `[ ]` package covers the need | **Recommend adoption.** Add as dependency, integrate, update checklist. |
| Package provides partial match | **Extend** the package with the missing capability. |
| No proprietary package matches, code is reusable | Plan as a **new proprietary package** from the start. |
| Code is strictly host-specific | Implement locally. Document the rationale. |

### 5. Record the Assessment
Add to the TODO:

```markdown
## Package-First Assessment
- Checklist consulted: Yes
- Relevant proprietary packages: <list or "none">
- READMEs read: <list>
- Decision: Use <package> / Adopt <package> / Extend <package> / New package <name> / Local implementation
- Rationale: <brief>
```

### 6. Post-Implementation
If a new proprietary package was created:
- [ ] Package has `README.md` following canonical format (`delphi-ai/templates/package_readme_template.md`)
- [ ] Run `bash delphi-ai/tools/verify_package_registry.sh` to update the checklist

## Validation
- Package-First Assessment is present in the TODO.
- If a proprietary package was found (used or available), the implementation leverages it.
- No new host-level utility files duplicate proprietary package capabilities.
- Checklist is current (re-run script if packages were added/removed).
